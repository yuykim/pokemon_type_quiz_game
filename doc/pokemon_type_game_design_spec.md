# 포켓몬 타입 상성 트레이닝 게임 — 상세 기획서 (개정판)

> 상성 데이터: `pokemon_type_matchup_notes.md` / `pokemon_type_matchup_notes.txt` (18타입, 2× / 0.5× / 0×)  
> 본 개정: 세션 문제 수, “둘 다” 출제 방식, 방어 입력 제한, **최적 선택지 전부 표시**, **타입별 UI 색상(HEX)** 반영

---

## 1. 제품 개요

### 1.1 한 줄 정의
플레이어가 **공격**·**방어** 상성을 연습하고, 제출 직후 **자신의 선택에 대한 배율**과 함께, **그 상황에서 더 나은(또는 동률 최선인) 모든 선택지**를 시스템이 계산해 보여 주는 **타입 상성 학습용 미니 게임**.

### 1.2 목표
- 공격 타입 하나 → 상대 타입(단일 또는 복합)에 대한 **최종 배율**을 익힌다.
- 들어오는 기술 타입(단일) 하나 → 내 방어 타입(단일 또는 서로 다른 복합)에 대한 **받는 배율**을 익힌다.
- 매 문제마다 **최적해가 무엇인지(복수면 전부)**를 확인해 판단력을 보완한다.

### 1.3 비목표 (초기 버전)
- 실제 포켓몬 도감·일러스트 의무 사용
- 특성, 날씨, 테라스탈 등
- 온라인 랭킹, 계정

---

## 2. 용어

| 용어 | 정의 |
|------|------|
| 기술 타입 / 공격 타입 | 플레이어가 고르거나 화면에 주어지는 **한 개**의 타입 (항상 단일) |
| 방어측 타입 | 피격 쪽. **1개** 또는 **서로 다른 2개** |
| 배율 | 복합 방어(또는 복합을 가진 상대)일 때 `table[기술][타입1] × table[기술][타입2]` (0이 한 번이라도 곱해지면 0) |
| 더 나은 선택 | 공격 문제에서는 **더 큰 배율**, 방어 문제에서는 **더 작은 배율**(받는 피해 관점에서 유리) |

## 3. 시작 화면 — 옵션 (고정값은 세션 동안 유지)

### 3.1 상대(방어측) 타입 모드 — 공격 문제 출제에만 사용
- **단일 타입만**: 공격 문제에서 상대는 항상 타입 1개.
- **복수 타입 허용**: 공격 문제에서 상대는 서로 다른 타입 2개 조합으로 출제될 수 있음.

### 3.2 연습 범위
- **공격만**: 모든 문제가 공격 문제.
- **방어만**: 모든 문제가 방어 문제.
- **둘 다**: §7대로, **문제마다** 공격 또는 방어 중 **하나만** 랜덤 출제.

### 3.3 세션 문제 수 **N** (**확정: 플레이어가 직접 정함**)
- 시작 화면에서 **이번 세션 문제 개수 N을 플레이어가 스스로 지정**한다 (숫자 입력 필드 등). 고정 프리셋(5·10·20만)에 제한하지 않는다.
- 구현 시 **유효 범위**만 두면 된다 (예: 최소 1, 최대 상한은 성능·UX상 합리적 값으로 캡).
- 세션은 **정확히 N문제**를 진행한 뒤 종료(또는 “한 번 더”로 재설정).

### 3.4 UI
- 옵션 A(단일/복수), 옵션 B(공격만/방어만/둘 다), **N**이 모두 확정된 뒤 **시작**으로 본 연습 화면 진입.

## 4. 데이터·계산

### 4.1 타입 집합 (18)
노말, 불꽃, 물, 풀, 전기, 얼음, 격투, 독, 땅, 비행, 에스퍼, 벌레, 바위, 고스트, 드래곤, 악, 강철, 페어리.

### 4.2 상성 테이블
- `(공격타입, 방어타입)` → `2`, `0.5`, `0` (UI: `2×`, `½×`, `0×` 등).
- 문서의 공격 기준 표를 코드화.

### 4.3 배율 함수
```
M(attack, [d1])     = T[attack][d1]
M(attack, [d1,d2])  = T[attack][d1] * T[attack][d2]   // d1 ≠ d2
```

## 5. 공격 모드 (문제 단위)

### 5.1 출제
- 옵션에 따라 상대(방어측) 타입 표시: 단일 1개 또는 서로 다른 2개.

### 5.2 입력
- 18타입 중 **1개** 선택 (클릭 또는 드래그 등).
- **공격** 버튼으로 확정.

### 5.3 결과
- 선택한 기술 타입에 대한 **최종 배율** 표시.
- (선택) 타입별 기여: `×2`, `×½` 등 분해 표기.

### 5.4 **더 나은 선택지 — 전부 표시 (공격)**
- 주어진 방어 타입(들)에 대해, 18개 기술 타입 각각의 배율을 계산한다.
- **최대 배율** `M_max`를 구한다.
- **`M_max`와 동일한 배율을 내는 모든 기술 타입**을 나열한다 (동률 최적해 전부).
- 플레이어 선택이 그 목록에 포함되면: “**이미 최선 중 하나입니다**” 등으로 안내 가능.
- 포함되지 않으면: “**더 유리한 기술 타입(전부)**: …” 형태로 위 목록 전체를 보여 준다 (한 타입만 최적이면 1개만 표시).

---

## 6. 방어 모드 (문제 단위)

### 6.1 출제
- **들어오는 기술 타입** 1개 표시 (항상 단일).

### 6.2 입력
- 18타입 중 **1개만** 고르거나 **서로 다른 2개**를 고른다.
- **`물/물`처럼 동일 타입을 두 슬롯에 넣는 것은 불가** — UI에서 선택 불가 또는 검증 오류 처리.
- 타입 **1개만** 선택한 경우: 실제 게임의 **단일 타입 포켓몬**과 동일하게 `M(기술, [d])`만 사용한다 (**확정**).

### 6.3 확정
- **방어** 버튼.

### 6.4 결과
- `M(기술, 선택한 방어 타입들)` 표시.

### 6.5 **더 나은 선택지 — 전부 표시 (방어)**
- 가능한 플레이어 선택 공간:
  - **단일**: 타입 18가지.
  - **복합**: 서로 다른 순서쌍 `(a,b)` where `a ≠ b`. 표시 시 `(a,b)`와 `(b,a)`는 동일 배율이므로 **한 가지 규칙으로 정규화**(예: 가나다순 첫 타입 우선)하여 **중복 없이** 최적 목록에 넣는다.
- 각 후보에 대해 `M(기술, …)`를 계산하고, **최소 배율** `M_min`을 구한다 (받는 피해 최소 = 방어 관점 최적).
- **`M_min`과 같은 배율인 모든 후보(단일+복합)**를 전부 나열한다.
- 플레이어 조합이 최적 집합에 들어 있으면 “**이미 최선 중 하나입니다**” 등.

---

## 7. “둘 다” 모드 — 세션·출제 규칙 (**확정**)

### 7.1 문제 구성
- 세션 문제 수는 **§3.3의 N**.
- **각 문제(1~N)마다** 난수로 **공격 문제** 또는 **방어 문제** 중 **정확히 하나**를 출제한다 (고정 순서 없음).

### 7.2 옵션 적용
- **공격으로 뽑힌 문제**: §3.1의 단일/복수 설정을 그대로 적용해 상대 타입을 생성.
- **방어로 뽑힌 문제**: 항상 기술 타입 단일 출제; §6의 입력·최적해 표시 규칙 적용.

### 7.3 세션 종료 화면 (권장)
- N문제 완료 후, 선택 사항으로 **요약**: 예) “공격 문제 정답(최선 선택) 비율”, “방어 문제 최선 비율” 등. (구체 지표는 구현 시 단순화 가능.)

## 8. UI/UX

### 8.1 타입 그리드
- 18칸, **§9 타입 색상**을 배경 또는 테두리에 사용.
- **텍스트 라벨 필수**(색각 이슈 대비).
- 전기·노말 등 밝은 배경은 **어두운 글자**, 어두운 타입은 **밝은 글자** 권장(§9 참고).

### 8.2 슬롯
- 공격: 선택 슬롯 1칸.
- 방어: 슬롯 2칸; **같은 타입 두 칸 채우기 불가**.

### 8.3 최적해 영역
- 긴 목록일 수 있으므로 **스크롤 가능한 패널** 또는 “접기/펼치기” 권장.
- 동률 최적이 매우 많을 때(특히 방어 쪽)도 **전부 표시**가 요구사항이므로, 성능상 문제 없음(최대 후보 수: 공격 18, 방어 18+153=171).

## 9. 타입별 대표 색상 (UI용 HEX)

### 9.1 출처·주의
- 닌텐도/게임프리크가 **공식 HEX 팔레트를 공개한 문서**는 일반적으로 제공되지 않는다.
- 아래 값은 **위키·팬 사이트·CSS 예제에서 널리 인용되는 “타입 아이콘” 계열 색**에 맞춘 것으로, **본편 UI와 1:1 일치를 보장하지는 않는다**. (참고: Bulbapedia `Help:Color templates`, 커뮤니티 정리 글·Gist 등에서 유사 값이 반복 인용됨.)
- 상업 서비스 시에는 자사 브랜드 가이드에 맞춰 조정 가능.

## 10. 난이도·출제 (권장)

- N이 크면 랜덤만으로 충분; 필요 시 **최근 출제 타입 중복 완화**.
- 복합 방어 최적 목록이 길어지는 경우가 자연스러우므로, **“왜 이렇게 많이 최적인가”**를 학습용으로 받아들이거나, UI에서 타입별 기여 분해를 병행 표기.

---

## 11. 기술 구현 메모

- 상성·색상·문자열은 각각 **단일 소스**에서 로드.
- 최적해 계산:
  - 공격: `max over 18 attacks` + filter `== max`.
  - 방어: `min over 18 + 153 combos` + filter `== min`, `(a,b)` 정규화.
- 단위 테스트: 단일/복합, 0× 포함, 동률 최적 여러 개인 케이스.

---

## 12. 저작권

- “Pokémon” 및 공식 자산은 각 권리자 소유. 타입 **이름·상성 규칙**은 게임 메카닉에 가깝고, **색상 HEX**는 커뮤니티 관행에 따른 UI 선택이며 공식 팔레트 선언이 아님을 UI/크레딧에 명시하는 것이 안전하다.
